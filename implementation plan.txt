LuminaQuest - Clean 3-Iteration Plan

Iteration 1 - Playable Story Engine MVP (Stable)
Objective:
Deliver an end-to-end vertical slice where admins can publish authored stories and players can complete sessions through LLM-mapped actions.

Scope checklist:
[x] Monorepo workspace setup (`server`, `web`)
[x] Dependency installation and runnable scripts
[x] Express + Mongo models + JWT auth (`admin`, `user`)
[x] Admin game CRUD + publish validation
[x] Player session engine (start, action resolve, status transitions)
[x] Async LLM resolver using OpenRouter with free-model support (`openrouter/free` or `:free`)
[x] Mock OpenAI Responses-style fallback when provider key/call is unavailable
[x] Minimal Admin Forge UI and Player Journey UI
[x] Add smoke-test verification notes and known constraints
[x] Finalize README iteration status and entrypoints
[x] Stability pass: Mongo retry + diagnostics + docker compose bootstrap
[x] Stability pass: safer route validation for ObjectId inputs
[x] Confirm persistence boundary: backend-only DB operations

Acceptance target:
An admin creates and publishes a 3-scene game, and a user plays from start to terminal with points/turn tracking and persisted history.

------------------------------------------------------------

Iteration 2 - Narrative Intelligence + Observability (Stable + Hardened)
Objective:
Improve narrative quality and trust while keeping server-deterministic outcomes.

Scope checklist:
[x] Wildcard policy service (bounded dynamic bridge rules)
[x] Two-stage resolution pipeline (intent map + narration layer)
[x] Provider fallback metrics and quality counters
[x] Trace hooks for resolver pipeline with optional Langfuse sink
[x] Admin graph visualization and graph analysis
[x] Draft playtest mode from arbitrary scene
[x] Player replay timeline and route explanation surfaces
[x] Security hardening (helmet, rate limits, input sanitization)
[x] Error handling hardening (async wrappers, standardized API errors, service logging)
[x] Frontend reliability hardening (error boundary, safe storage, side-effect cleanup)
[x] Maintainability hardening (split components, refactor session handler, indexes)
[x] Production baseline (tests, CI workflow, Dockerfiles, API docs, /api/v1 alias)

Acceptance target:
Creative inputs can route through server-approved wildcard bridges without introducing invalid scenes or rewards.

------------------------------------------------------------

Iteration 3 - 8-Bit Visual Presentation Layer (Delivered Core + Local Provider Switch)
Objective:
Transition from basic chat card gameplay to layered 8-bit story staging while preserving backend authority.

Scope checklist:
[x] SceneRenderConfig and visual delta support in backend session updates
[x] Layered stage rendering components in React
[x] Pixel-style transition pipeline tied to avenue outcomes
[x] Environment/sprite state persistence across turns
[x] Ending grading and grading panel states
[x] Config-driven provider switch across external + local (`openrouter`, `lmstudio`)
[ ] Docker build verification blocked in current sandbox (daemon unavailable)
[ ] Native MLX adapter (future)
[ ] Native WebGPU adapter (future)

Acceptance target:
Player actions trigger visible world-state changes in an 8-bit presentation, synchronized with deterministic backend updates.
